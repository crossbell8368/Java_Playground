* 부분합 문제에 대한 설명으로 옳은것은

1. 원소 갯수가 10만개이고 S의 값이 최대 1억이기에 부분합을 구할때 int가 아닌 long을 사용해야 한다. (O)
    => int 자료형의 범위는 대략 21억
    => S 최대값이 1억 이라면, 부분합은 21억을 넘어설 수도 있음
    => 이에 넉넉한 long 자료형을 사용하는 것이 타당

2. 수열에서 연속된 수들의 부분합 중에 그 합이 S가 되는것 중 가장 짧은 것의 길이를 구하는 문제이다. (O)
    => 문제와 동일하게 맞음

3. S와 같은 값을 가지는 수열의 원소가 있다면 정답은 1이다. (0)
    => 투 포인터가 동작하던 중, S값과 같은 값을 탐지하면 해당 값으로 포인터가 모아질 것이기에 1

4. 수열의 모든 숫자를 더해도 S보다 작을 수 있다. (0)
    => 수열의 모든 숫자를 더해도 S보다 작은 것은 0을 출력하는 하나의 경우가 될 것
