
1. 정점의 갯수 N이 최대 1000 이기 때문에, O(N^2) 의 공간복잡도가 필요한 인접행렬로도 그래프를 표현할 수 있다. (O)
    => 인접행렬로 구축할 경우, 최대 사이즈는 1,000,000 정도이기에, 표현은 가능하다.

2. 간선의 갯수 M에 대하여 출력되는 탐색 수행 결과, 가능한 방문 정점이 최소가 되는 입력은 모든 간선이 특정 정점을 공유하는 것으로 (X)
  이때 출력되는 방문 정점의 개수는 M+1 개 이다.
    => 하나의 정점이 중심이 되어, 다른 모든 정점들과 연결되어있는 중앙집권식 구조를 의미하는건가..?
    => 이떄 정점의 개수는 M - 1 개이지 않은가..?

3. 예제 입력 3처럼 출력되는 방문 순서는 입력에 주어진 연결된 정점만 포함하며, 입력에 주어졌다면 항상 방문할 수 있다 (O..?)
    => 예제 입력 3은, 수많은 정점과 하나의 간선만 주어진 케이스
    => 그렇기에 방문 순서는 '입력정점 -> 연결된 정점' 으로 구성될 것
    => 이러한 구성으로 본다면 입력으로 주어진 정점은 곧, 유일한 연결점..? 을 의미하는 것으로 보여짐
    => 그렇기에 항상 방문 가능하다고 보여짐

4. 같은 그래프에 대하여, DFS를 수행한 결과와 BFS를 수행한 결과로 출력된 정점의 갯수는 항상 같다. (O)
    => DFS(깊이우선) 방식과 BFS(너비우선) 방식은 모두 모든 정점을 확인하는 방식이다.
    => 그렇기에 방문 정점의 순서는 차이가 날 수 있으나, 정점 갯수에는 차이가 없을 것
